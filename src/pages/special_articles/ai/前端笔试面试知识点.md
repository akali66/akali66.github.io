## 第一部分 html

首先我认为，笔试面试的核心在你对框架的认识，html可能仅仅只会占一小部分，或者只是提一嘴，所以我准备大致讲一些核心或常见知识点：

**html的本质**

你可以理解为一棵“语义树”（DOM树），每个标签（元素）是树上的一个节点，是有特定含义的

那`<div>`和`<span>`就是属于“无意义”的通用节点

**语义化**

在HTML5之前，写代码就是用多个`<div>`来布局，如`<div id="header">`、`<div class="nav">`，对机器（搜索引擎、屏幕阅读器）来说，其实都是一堆没有任何意义的div

而语义化标签的引入，就让结构本身有了意义，等于每个标签的“人设”

常用的语义化标签有（可以先简单过一下）：

- `<header>`：页面或章节的页眉。通常包含网站Logo、主导航、搜索框等。注意，一个页面可以有多个 `<header>`，比如文章内也可以有一个小的 `<header>`包含文章标题和元信息。
- `<nav>`：主导航区域。包含一组重要的链接，用于跳转到站内其他页面（如顶部导航、侧边栏导航）。不应将所有链接组都放入 `<nav>`，只用于主要导航区块。
- `<main>`：页面的核心、独立内容。每个页面有且仅有一个 `<main>`。其内容应是该页面独有的，不应包含全局性、重复出现的内容（如导航、页脚）。
- `<article>`：独立、完整、可独立分发的内容块。比如一篇博客文章、一条论坛帖子、一则新闻、一个独立的微件。关键在于，把它单独拿出来放到另一个上下文中，它仍然是完整、有意义的。
- `<section>`：文档中的一个主题性分组。通常带有一个标题。它用于将相关内容组织在一起。与 `<article>`的区别是，`<section>`的内容不一定能独立存在。比如，一本书的每一章可以用 `<section>`，而书中的每一篇文章则用 `<article>`。
- `<aside>`：与主要内容间接相关的内容。比如侧边栏、批注、广告、相关文章列表等。它应独立于其周围的内容。
- `<footer>`：页面或章节的页脚。通常包含版权信息、联系方式、相关文档链接等。

**为什么推荐用语义化标签？**
1. 对开发者友好：简单，易读，有规范。很好理解
2. 对SEO友好：搜索引擎的爬虫会分析页面结构，看到（`<main>`,`<article>`,`<h1>`）能帮爬虫更准确的理解内容和权重关系
3. 对无障碍访问友好：屏幕阅读器用户依赖这些标签来理解页面结构

**元信息**

讲完整体的，其中在构建html的时候就会遇到`<head>`标签里存在的元信息，它们是不显示但及其重要的，但是目前用的不多，所以大致提几个：

- `<meta charset=“UTF-8”>`：指定文档字符编码，避免乱码。必须放在 `<head>`的最前面
- `<meta name=“viewport” content=“width=device-width, initial-scale=1.0”>`：移动端适配的基石。它告诉浏览器视口宽度等于设备宽度，并且初始不缩放。


**可访问性(a11y)**

可访问性不是可选项，而是构建现代Web应用的责任。HTML 是 a11y 的第一道防线。(官方的话，不是我的话)

比较常见的就是
`alt`：`<img>`的替代文本。不仅是SEO，更是为视障用户描述图片内容。如果图片是装饰性的，可以写 alt=“”，但绝不能省略。

**ARIA 属性**：当 HTML 语义不足以描述交互时，用 ARIA 来补充。
- `role`：定义元素的角色，如 role=“button”、role=“dialog”、role=“navigation”
- `aria-label`：为元素提供一个不可见的标签。当元素本身没有文本内容时特别有用。

```javascript
<button aria-label="关闭弹窗" onclick="closeModal()">X</button>
<!-- 屏幕阅读器会读出“关闭弹窗 按钮”，而不是“X 按钮” -->
```

- `aria-labelledby`：通过ID关联到另一个元素，将其作为自己的标签

这些大致就是html需要提出来讲的点，那复习完这几个点后，可能在面试中还会问到你一些别的内容：

1. `<!DOCTYPE html>`的作用是什么？
- 它不是标签，是为了声明。告诉浏览器我这个网页是遵循HYML5规范的。写了这个，就会触发浏览器的**标准模式**，而不是**怪异模式**，所以它必须放在HTML文档的第一行

2. 行内元素和块级元素的根本区别是什么？
- 首先他们两个在布局方式、内容模型上都有不同：
- **块级元素**独占一行，从上到下垂直排列，宽度默认填满父元素，可设置宽度高度内外边距。如`<div>`,`<p>`,`<h1>`,`<ul>`,`<li>`
- **行内元素**不独占一行，排在同一行直到排不下再换行，其宽度、高度、上下外边距、上下内边距设置无效。宽度和高度由其内容决定，如`<span>`、`<a>`、`<strong>`、`<em>`、`<img>`、`<input>`
- 内容模型中，块级元素通常可以包含其他块级元素和行内元素。行内元素通常只能包含数据或其他行内元素（例如，`<a>`里不能再放一个 `<a>`）。
- display属性：可以通过CSS的 display属性改变其默认行为（如 display: inline-block;使其兼具两者特性）。

3. src和 href的区别？
这其实是一个关于资源加载行为的问题
- `src(Source)`：表示“引入资源到当前文档中”，当解析到带src（如 `<script>`、`<img>`、`<iframe>`）的元素，则**会暂停其他资源的下载和处理**，直到该资源加载、解析、执行完毕。理解为“嵌入”关系
- `href(Hypertext Reference)`:表示“当前文档与外部资源之间建立链接”。浏览器在解析到带有 href的元素（如 `<link>`、`<a>`）时，会并行下载该资源，不会停止对当前文档的处理。它是一种“关联”关系。

4. 前端性能优化中，HTML 层面能做什么？
这个是一种分析题，首先你得从想象浏览器的加载，会经过哪些过程，如果不知道，后面会讲到，这里暂时过一下：
- 首先浏览器加载会渲染DOM树，那么最直接的优化方法就是尽可能的减少DOM树
那么渲染时就会涉及到html，css和js的加载，那同样可以根据这个点来思考一下三点：
- 多使用语义化标签，有助于浏览器更高效解析
- 将CSS放在头部，可以避免页面渲染后再应用样式，导致样式闪烁
- js放在底部（`<body>`之前），或使用async/defer属性，来避免js阻塞html解析和页面渲染
相对你可以相到的一个更好的点是：
- 利用 preload/prefetch：`<link rel=“preload”>`用于提前加载当前页面必定会用到的关键资源（如首屏字体、关键CSS/JS）。`<link rel=“prefetch”>`用于提前加载下一个页面可能用到的资源。

## 第二部分 JS

一般的流程是讲css，但是我认为，这个时候有必要先把js拿出来讲，因为个人感觉css大致都是一些样式上的知识点，这不都是对着设计图敲嘛？所以暂时先放后

那最基本的点就是ES6引入的内容了：

**变量声明：var, let, const 到底有什么区别？**

1. `var`最古老的，问题最多,第一想到的就是函数作用域和变量提升
- **作用域**：函数作用域，在整个函数内有效
```javascript
function testVar() {
    if (true) {
        var a = 10; // 在if这个块里声明的
    }
    console.log(a); // 输出 10,因为var是函数作用域，在函数内都能访问到。
}
testVar();
```
- **变量提升**：`var`声明的变量会被“提升”到其所在作用域的顶部。注意，提升的是声明，不是赋值。
```javascript
console.log(myVar); // 输出 undefined，而不是报错
var myVar = 5;
// 实际执行顺序被JS引擎理解为：
// var myVar; // 声明被提升到最前面
// console.log(myVar); // 此时myVar是undefined
// myVar = 5; // 赋值留在原地
```
- 可以重复声明：`var a = 1; var a = 2;`不会报错，这容易导致代码错误。

2. `let`和`const`(ES6引入，现代写法)
相对比之下对应的是：块级作用域、变量提升？（暂时性死区），不可重复声明
- **块级作用域**：块级作用域。{}大括号内就是一个块，如 `if`、`for`、`while`或单独的大括号。
```javascript
if (true) {
    let b = 20;
    const c = 30;
}
console.log(b); // 报错：b is not defined。因为b只在if这个块里有效。
console.log(c); // 报错：c is not defined。
```
- **变量提升？**：他们其实也有提升，但是和var不一样的是，声明之前会报错，叫做“暂时性死区”
```javascript
console.log(myLet); // 报错：Cannot access 'myLet' before initialization
let myLet = 5;
```
- 不可重复声明：在同一作用域内，`let a = 1; let a = 2;`会报语法错误。

对于`let`vs`const`:
- `let`用于声明值可变的变量。
- `const`用于声明常量。**它保证的是变量指向的内存地址不变**，而不是值不变。
    - 对于基本类型（数字、字符串、布尔等），值就保存在那个地址里，所以 `const PI = 3.14; PI = 3.15;`会报错。
    - 对于引用类型（对象、数组），地址不变，但对象内部的内容可以改变。
```javascript
const person = { name: '小明' };
person.name = '小红'; // 这是允许的,修改对象内部属性。
console.log(person); // { name: '小红' }

person = { name: '小刚' }; // 报错,试图给person赋一个新的对象（新地址）。
```

那其实总结下来，你要知道的点就是：首先，var是函数作用域，且是有变量提升，可以重复声明的，是历史遗留的问题。而let和const是块级作用域，有暂时性死区，不可重复声明。而const用于声明常量，保证其引用不变（再深入解释一下）,而现在基本上默认都是使用const，只有当你知道这个变量需要被重新赋值的时候，就会用let，基本不用var

**闭包**

js的重点，也是笔试面试的重点，很多都会考到

那想了解闭包，就先了解一下作用域：作用域是变量和函数可以被访问的范围，可以理解为，一个房子，你在你自己房间（局部作用域）放的东西,在客厅（全局作用域）的人不一定能直接拿到

那闭包就是：**当一个函数可以记住并访问其所在的词法作用域，即使这个函数在其词法作用域之外执行，就产生了闭包。**

简单来说：函数“包”住了外面的变量

举个例子：
```javascript
function createCounter() {
    let count = 0; // count 是 createCounter 函数的局部变量
    return function() {
        count++;   // 内部函数访问了外部函数的变量count
        console.log(count);
    };
}
const counter = createCounter(); // createCounter执行完，按说count该被销毁了
counter(); // 输出 1
counter(); // 输出 2
counter(); // 输出 3
```

内部函数一直保持这对`count`的引用，垃圾回收机制不会回收它，这个`count`变量就变成了一个私有变量，只能通过`counter`函数来修改

那这么看闭包会使没有用还扫不到的垃圾越来越多，那它有什么用呢？

闭包是函数和其周围状态（词法环境）的引用捆绑在一起。它的主要用途是创建私有变量和函数，常用于模块化、防抖节流等场景。

**this**

重要考点之一，四大绑定规则
- 默认绑定：独立函数调用。在非严格模式下，`this`指向全局对象（浏览器中是window）；严格模式下是`undefined`。
```javascript
function foo() { console.log(this); }
foo(); // 浏览器中输出 undefined (严格模式)
```
- 隐式绑定：函数作为对象的方法被调用。this指向调用它的那个对象。
```javascript
const obj = {
    name: ‘前端’,
    sayName: function() { console.log(this.name); }
};
obj.sayName(); // 输出 ‘前端’，this指向obj
// 坑：丢失绑定
const fn = obj.sayName;
fn(); // 输出 undefined，因为现在是独立调用，this指向全局（或undefined）
```
- 显式绑定：用 `call`, `apply`, `bind`强行指定`this`
```javascript
function greet() { console.log(`Hello, ${this.name}`); }
const person = { name: ‘Alice’ };
greet.call(person);   // 输出 Hello, Alice
greet.apply(person);  // 同上，传参方式不同
const boundGreet = greet.bind(person); // bind返回一个新函数
boundGreet();         // 输出 Hello, Alice
```
- `new`绑定：用`new`关键字调用构造函数。`this`指向新创建的那个实例对象。
```javascript
function Person(name) {
    this.name = name; // this指向新创建的对象
}
const p = new Person(‘Bob’);
console.log(p.name); // Bob
```
- 箭头函数：没有自己的`this`。它的`this`继承自定义时所在的外层普通函数的`this`，且一旦绑定就无法更改。
```javascript
const obj = {
    name: ‘obj’,
    regular: function() {
        console.log(this.name); // 指向obj
        const arrow = () => {
            console.log(this.name); // 继承自regular的this，也指向obj
        };
        arrow();
    },
    arrowFunc: () => {
        console.log(this.name); // 定义在对象字面量中，外层是全局，this指向window
    }
};
obj.regular(); // 输出 ‘obj’, ‘obj’
obj.arrowFunc(); // 输出 ‘’ (window.name)
```

所以整体规则就是：先看函数怎么被调用。1. new调用 -> 新对象。2. call/apply/bind调用 -> 指定的对象。3. 对象方法调用 -> 该对象。4. 普通函数调用 -> 全局（或undefined）。箭头函数看外层。
