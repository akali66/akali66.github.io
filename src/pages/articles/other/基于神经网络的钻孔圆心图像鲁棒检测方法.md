# 2.专有名词

## 2.1 AdamW

- 是什么：训练时更新参数的优化器。
- 为什么在本项目用它：中小数据集场景下往往更稳，泛化更好。
- 如何配套：和学习率衰减一起使用。

```python
# ================== 核心优化与高级超参数 (这是你的论文卖点) ==================
optimizer='AdamW',             # 优化器: 使用目前学术界最主流、收敛更稳定且抗过拟合最好的 AdamW
lr0=0.001,                     # 初始学习率 (Learning Rate): 决定模型学习的跨度步伐
lrf=0.01,                      # 最终学习率衰减因子 (lr0 * lrf)
patience=50,                   # 早停机制 (Early Stopping): 如果连续 50 轮指标毫无提升，自动停止，防止模型死记硬背(过拟合)
```

AdamW 负责稳定更新，lr0 和 lrf 控制前快后稳，patience 防止无提升时继续过度训练。

## 2.2 数据增强

```python
# ================== 高级数据增强策略 (提升论文要求的"鲁棒性") ==================
mosaic=1.0,                    # 开启马赛克增强 (把4张图拼成1张，极大提升复杂背景和遮挡的识别能力)
mixup=0.1,                     # 图像混合增强 (进一步提升模型对泥浆等遮挡物的泛化)
hsv_h=0.015, hsv_s=0.7, hsv_v=0.4, # HSV 颜色空间随机干扰 (模拟真实的矿井/钻孔各种光照、过曝和极暗的情况)
degrees=10.0,                  # 随机旋转 (-10 到 10度，模拟探头旋转拍摄带来的倾斜)
flipud=0.5,                    # 随机上下翻转 (模拟探头上下移动的各种倒置视角)
fliplr=0.5,                    # 随机左右翻转的概率
```

增强不是“乱改图”，而是模拟真实干扰，让模型对遮挡、光照和姿态变化更鲁棒。

# 3 附加点

Q1：为什么要做四种方法，不直接只用 YOLO？

A：四种方法针对不同场景互补。传统法可解释、计算快；分割法在复杂背景更稳。工程上“可解释 + 泛化”比单模型更实用。

Q2：核心创新是什么？

A：不是单模型刷分，而是统一接口下的多算法可比体系，以及分算法适配指标设计。

Q3：如何保证结果可复现？

A：训练参数、日志、权重、曲线都在 runs 目录固化保存，脚本可重复执行。

Q4：AdamW 和 Adam 的差异你怎么解释？

A：AdamW 对权重衰减处理更清晰，实际训练更稳。我的项目中它和学习率衰减、早停是组合策略。

Q5：为什么指标不统一用 mAP？

A：在线服务里比较的是“圆心与圆拟合质量”，不同算法输出形态不同，单一指标会失真，所以分流更公平。

Q6：YOLO 的圆心计算为什么提供两种方法？

A：不同噪声形态对中心计算敏感性不同。图像矩法抗不规则，外接圆法抗边缘残缺。

Q7：如果被提出说数据量偏小？

A：承认现阶段规模，并说明已通过增强和对比实验验证方法有效，下一步是扩充外场数据与跨设备验证。

Q8：如果被提出只是调参数？

A：参数可调是工程必要条件，但目前实现的是完整系统设计：统一接口、可比指标、可复现训练链路和可导出报告。

Q9：系统最难点是什么？

A：统一不同算法的输出口径并建立公平评价体系，这比单一模型调参更难。

Q10：你的结果能直接上生产吗？

A：当前是可部署原型，已具备基础能力，仍需做更多现场域数据验证和时序稳定优化。
