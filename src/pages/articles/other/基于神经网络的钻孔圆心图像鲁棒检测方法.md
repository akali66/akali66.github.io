# 基于神经网络的钻孔图像鲁棒检测方法

## 1. 系统架构与请求全流程 (前端 $\rightarrow$ 后端)

### 1.1 前端：参数收集与数据发送
**代码位置**：`index.html`
当用户在浏览器拖拽图片并点击“开始检测”时，前端是如何打包数据的？
* **获取输入与参数**：前端记录了用户勾选的算法(`methodSelect`)，是否是对比模式(`compareMode`)，以及右侧动态生成的滑块参数集合 `currentParamState`。
* **构建 FormData发送**：
  ```javascript
  const formData = new FormData();
  formData.append('file', currentFile); // 二进制图片文件
  formData.append('method', ui.methodSelect.value); // 指定哪个算法
  formData.append('compare_mode', ui.compareMode.checked); // 布尔值对比模式
  formData.append('params', JSON.stringify(currentParamState)); // 将滑块字典转成字符串
  // 通过 fetch API，以 POST 请求打向后端的 /api/detect
  const res = await fetch('http://127.0.0.1:8000/api/detect', {...});
  ```
> **💡术语解释——FormData/POST**：浏览器标准的上传协议包裹格式，就像寄快递前要把图片数据、文字参数分类打包进一个快递盒，并贴上 POST 面单发往后端接口地址。

### 1.2 后端：接收并解流图片
**代码位置**：`app.py` (`detect_image`函数)
* **将二进制流还原为图像**：
  ```python
  contents = await file.read() # 拿到前端的二进制乱码
  nparr = np.frombuffer(contents, np.uint8) # 转换为一维的数字阵列
  image = cv2.imdecode(nparr, cv2.IMREAD_COLOR) # 重新解码为3通道彩色 ndarray 图像矩阵
  gray_image = cv2.cvtColor(image, cv2.COLOR_BGR2GRAY) # 转成灰度图（单通道），以备大多数机器视觉算法使用
  ```
> **💡术语解释——灰度图与ndarray矩阵**：计算机看图片其实是看 Excel 表格。彩色图是三层表格(B, G, R)，计算量大且受光斑颜色干扰；灰度图被压成单层表格，数字只有 0(纯黑) 到 255(纯白)，计算机只需关注“颜色的明暗交界”，就能找出边缘。

### 1.3 后端：算法路由管线 (字典调度)
后端如何知道该运行那套代码？
```python
methods_dict = {
    "method1": (detect_hough, parsed_params.get("method1", {})),
    "method2": (detect_min_enclosing, parsed_params.get("method2", {})),
    "method3": (detect_canny_hough, parsed_params.get("method3", {})),
    "method4": (detect_yolo_segmentation, parsed_params.get("method4", {}))
}
# 如果是对比模式，就四个全部依次执行；否则只执行前端指定的方法
methods_to_run = list(methods_dict.keys()) if compare_mode else [method]
for method_id in methods_to_run:
    func, current_params = methods_dict[method_id]
    res = func(gray_image, current_params) # 开始跑核心算法
```
> **💡术语解释——路由字典机制扩展性**：一种优秀的设计模式，把算法名字和真正的执行函数(`func`)绑定。未来如果学分弟开发了 method5，只需往字典里加一行，主流程完全不用改，这在软件工程中叫做“解耦”。

## 2. 算法一：霍夫圆变换 (Hough Circle Transform)
**代码位置**：`detectors.py` $\rightarrow$ `detect_hough`

### 2.1 算法的核心数学逻辑
霍夫圆并不是“把圆画出来比对对”，而是使用**参量空间投票机制**。标准圆的方程是：$(x-a)^2 + (y-b)^2 = r^2$。
算法会在找到的“边缘像素点”处，朝着法线（梯度的垂直方向）射出一条线，在线上可能成为圆心 $(a,b)$ 的位置都投入一张“信任票”。如果有大量的边缘点都不约而同地把票投向了同一个点坐标，这个点的票数就会爆表——它就是圆心！

### 2.2 详细代码步骤与参数
1. **降噪模糊 (高斯滤波)**：
   ```python
   blurred = cv2.GaussianBlur(gray_image, (5, 5), 0)
   ```
   > **💡术语解释——高斯滤波 (Gaussian Blur)**：把每个像素和周围像素的颜色求“带权重的平均值”，就像人眯着眼睛看图。这样能把煤层上微小的麻点、噪点“揉散”，防止它们被误认为是坑洞边缘。这属于数据预处理。(5,5)代表视野窗口大小。

2. **算法执行**：
   ```python
   circles = cv2.HoughCircles(
       blurred, 
       cv2.HOUGH_GRADIENT,  # 使用基于梯度的霍夫(Hough)算法
       dp=dp, minDist=minDist, param1=param1, 
       param2=param2, minRadius=minRadius, maxRadius=maxRadius
   )
   ```
   * **`dp` (累加器分辨率)**：为 1.0 时系统投票地图尺寸与原图1:1；如果设为 2.0，投票地图的宽高缩减一半。这能容忍圆不够完美，也就是票随便偏一点也能落在同一个格子里，但精度会下降。
   * **`minDist` (圆心距离)**：强行要求这片区域内只能“当选”一个圆，防止同一个孔提取出多个稍微偏移的圆。
   * **`param1` (内部Canny高阈值)**：算法底层自己做边缘寻找时的“要求严苛度”。数值越高，找出的参考边缘越精确越少。
   * **`param2` (当选得票门槛 🌟最关键)**：即一个坐标点累计获得多少“票”才承认它是圆心。调低能查出残缺的圆，但容易把乱石错认成圆。

## 3. 算法二：轮廓最小外接圆 (Min Enclosing Circle)
**代码位置**：`detectors.py` $\rightarrow$ `detect_min_enclosing`

### 3.1 算法的核心数学逻辑
因为霍夫圆强制要求边缘是弧形的（要求梯度朝向圆心），但在极度恶劣的环境下，钻孔可能是个多边形/残破不规则洞穴。**方法二不再受几何代数的圆方程束缚**，而是像切蛋糕一样：先把图像按明暗切分出白色的板块（Blob），然后包络出一个恰好能把它全框进去的最小圈圈。

### 3.2 详细代码步骤与参数
1. **二值化与大津法切分**：
   ```python
   _, thresh = cv2.threshold(blurred, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
   ```
   > **💡术语解释——大津法 (Otsu's Method) 与二值化**：传统二值化是人工设定127——比它亮的全部变白（255），暗的全变黑（0）。大津法则是靠机器自己算方差，利用数学算出“哪一个数值能把图像完美分成明显反差的两拨黑白势力”。

2. **寻找区域轮廓连通块**：
   ```python
   contours, _ = cv2.findContours(thresh, cv2.RETR_LIST, cv2.CHAIN_APPROX_SIMPLE)
   ```
   > **💡术语解释——提取轮廓 (findContours)**：把二值化图上的白色连片区域一圈的坐标点收集起来。`RETR_LIST`代表只要有圈就捞，不考虑洋葱那样的层级嵌套结构。

3. **人工经验过滤 (消除伪影)**：
   提取后的框可能有一两百个，我们要根据常理过滤：
   * `min_area`（最小面积）：把小于某指甲盖大小的杂块抛弃。
   * `circularity`（圆度）：公式 $\text{Circularity} = 4\pi \times Area / Perimeter^2$。绝对正圆得分是 1。如果是个狭长的裂缝，圆度会低到 0.0X。用阈值淘汰裂缝干扰。

4. **使用矩 (Moments) 挑选最近目标与拟合**：
   ```python
   M = cv2.moments(c) # 形态矩
   cx, cy = int(M["m10"] / M["m00"]), int(M["m01"] / M["m00"])
   ```
   > **💡术语解释——图像矩 (Moments)**：高等数学积分在图像上的应用，可以理解为物体的“质量中心/重心计算”。算出了重心的 (cx, cy) 后，根据用户策略（靠近中心、最圆、面积最大）挑选唯一的唯一目标。然后再使用 `cv2.minEnclosingCircle` 求得完全包裹的坐标 $(x,y)$ 与 $Radius$。

## 4. 算法三：Canny边缘层自定义 + 单尺度霍夫 
**代码位置**：`detectors.py` $\rightarrow$ `detect_canny_hough`

### 4.1 算法动机：为什么还要手搓一个变体霍夫？
霍夫算法（算法一）是个黑盒，它底层自己抽骨架很容易断。一旦边缘残缺断裂，法线投票就投不中靶心。算法三相当于做了一台精密的“外科手术”，我们**手动帮它搭骨骼线，并将修补好的骨骼线生生喂给霍夫算法**。

### 4.2 详细代码步骤与参数
1. **外部使用自定义 Canny 过滤：**
   ```python
   edges = cv2.Canny(blurred, canny_low, canny_high)
   ```
   * `canny_high`（强边缘起点）与 `canny_low`（弱边缘延续点）：用高低两级门槛。高门槛确认这是边界起点，然后沿着起点向外爬延，只要爬到的像素高于低阈值，就承认它也是边界。

2. **神来之笔：形态学闭运算修补（连接断裂）🌟**
   ```python
   kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (5, 5))
   edges = cv2.morphologyEx(edges, cv2.MORPH_CLOSE, kernel)
   ```
   > **💡术语解释——形态学闭运算 (Morphological Close)**：就是“先膨胀后腐蚀”。想象边缘线条是一条有断裂裂缝的细面条。先“加粗”（膨胀），面条变粗后，缝隙互相碰到挤死了；然后再把周边均匀“削瘦回去”（腐蚀）。最终面条恢复原本的粗细，但缝隙被神奇地黏合了！

3. **欺骗性质的霍夫调用**：
   ```python
   cv2.HoughCircles(edges, cv2.HOUGH_GRADIENT, param1=1, ...)
   ```
   我们将提取好且粘好的纯白线 `edges` 放进去，并强行把 `param1` 置为 `1`，意思是：请霍夫算法放下防备完全信任这幅输入图纸！这样就把 Canny 调参和 票数（param2）彻底分离出来，增加了恶劣环境检测存活率。

## 5. 算法四：YOLOv8 深度学习实例分割
**代码位置**：`detectors.py` $\rightarrow$ `detect_yolo_segmentation`

### 5.1 算法逻辑
方法一二三都是“人类定规矩让机器找”。方法四是数据驱动：通过卷积神经网络成千上万次的矩阵乘法（AdamW优化训练出来的权重），模型已经“理解了孔洞语义”，面对黑影遮挡、半张图全是泥浆也能精准分割。

### 5.2 深度学习检测链路拆解
1. **加载模型 (使用单例模式防爆存)**：
   每次请求都读 pt 文件会导致服务器内存崩溃，所以采用全局变量缓存的模型池：
   ```python
   if _yolo_model is None: _yolo_model = YOLO(model_path)
   ```
2. **推理推断**：
   ```python
   rgb_image = cv2.cvtColor(gray_image, cv2.COLOR_GRAY2RGB) # 模型第一层写死了要求(B,G,R)3通道结构
   results = model.predict(rgb_image, conf=params.get('conf_thresh', 0.25))
   ```
   > **💡术语解释——置信度 (Confidence Threshold)**：模型对自己给出的结论打分！0.25代表如果模型觉得那团影子只有 24% 的概率是钻孔，它就选择闭嘴不圈出来。调低能漏检补救，但是会带来误报。
3. **将 Mask 面具转接回标准圆参数**：
   YOLO 的输出是一坨五颜六色的“掩码色块”。系统将其边缘抓取（用轮廓），然后再送入**图像矩中心计算**。从而将前端系统抹平，四个系统输出统统是 $[x,y,r]$ 统一格式。

## 6. 四大指标与度量系统 (如何保证公平对比？)
**代码位置**：`eval_metrics.py` 与 `app.py` (80~95行)
为什么答辩极光出众？因为你不仅调包，还写了标尺来度量它们。

1. **线覆盖率算法 (Edge Coverage) ✨**
   ```python
   dilated_edge_map = cv2.dilate(original_edge_map, kernel, iterations=1)
   hit_pixels = dilated_edge_map[ideal_circle == 1]
   ```
   按照它报出来的 $(x,y,r)$ 在空白图上画一个细若游丝的纯圆。看看这个理想的圈，跟图片中真实的粗糙边缘（膨胀后）压中了多少像素。这是终极裁决标准。由于 YOLO 提取的是宏观语义平滑圆，所以我代码里宽容地给予了 YOLO 算法 `tol=6` 像素的膨胀宽容带测试。

2. **霍夫独有评价：投票置信度 (Hough Confidence)**
   拿当选圆心在法线积累的总票数，除以绝对完美圆应该拿到的满票。分值越靠近 1 说明在光栅层面上拟合得越真。

3. **面提取独有评价：二维交并比 (Mask IoU)**
   对于算法二/算法四这种提取完整块颜色的。算出：(提取的掩码面积 $\cap$ 预测出的纯数学圆区域面积) $/ (两者相加去重的并集面积)$。这是工业界判断“不规格图形拟合饱满度”最核心的法则。

## 7. 神经网络进阶术语备忘（答辩必问预案）
* **AdamW 优化器**：用来训练更新深度学习模型“突触连接力度”（权重）的核心算法。Adam是自适应学习率，它跑得快但有时收敛不稳。末尾加了 "W" (Weight Decay) 分离出的权重衰减，能更好地防止机器在极少的数据下产生死记硬背（防止过拟合）。
* **数据增强技术**：对原始 160 张图做 色彩翻转、剪切、甚至随机遮盖。这不是为了作弊造假，而是**人为模拟极端恶劣地质工况**，强迫模型在最糟糕的情况下去“认特征”，从而极大地提升在工地现场抗干扰的鲁棒性。
* **AMP (自动混合精度)**：在显卡计算时，正常都用 32 位超高精度浮点数算，这极度耗费显存。AMP就是让部分对精度不敏感的网络层降级用 16 位算。最终实现**显存开销减半，训练速度翻倍**且准确率完全不衰减的高级工程手段。
